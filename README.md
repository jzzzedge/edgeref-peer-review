# EdgeRef AI Pre-submission Review

EdgeRef is a Streamlit-based AI-assisted pre-submission review tool for academic manuscripts. It helps authors check manuscript readiness, source data status, target journal fit, and revision priorities before formal journal submission.

> EdgeRef does **not** replace formal journal peer review. Its output is for pre-submission self-check and revision support only.

## Main Features

- Manuscript upload and parsing: PDF, DOCX, TXT
- Source data upload and parsing: CSV, XLSX, ZIP
- Target Journal Profile input
- Preflight Check with risk score and suggested actions
- AI-assisted peer review report generation
- Markdown and Word report export
- User-facing page hides API provider, model, and API key fields
- Footer branding: `Generated by EdgeRef`

## Run Locally

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Backend API Configuration

Ordinary users do **not** need to enter any API key on the page.

Administrators must configure the backend key through Streamlit Secrets or environment variables.

### Option 1: Local `.streamlit/secrets.toml`

Create `.streamlit/secrets.toml` in the project root:

```toml
DEEPSEEK_API_KEY = "your_key_here"
DEEPSEEK_MODEL = "deepseek-v4-flash"
EDGEREF_ACCESS_CODE = "your_access_code_here"
```

Do **not** commit `.streamlit/secrets.toml` to GitHub. It is already included in `.gitignore`.

### Option 2: Streamlit Cloud Secrets

In Streamlit Cloud, open **App settings → Secrets** and add:

```toml
DEEPSEEK_API_KEY = "your_key_here"
DEEPSEEK_MODEL = "deepseek-v4-flash"
EDGEREF_ACCESS_CODE = "your_access_code_here"
```

### Option 3: Environment Variables

```bash
export DEEPSEEK_API_KEY="your_key_here"
export DEEPSEEK_MODEL="deepseek-v4-flash"
export EDGEREF_ACCESS_CODE="your_access_code_here"
```

Windows PowerShell:

```powershell
$env:DEEPSEEK_API_KEY = "your_key_here"
$env:DEEPSEEK_MODEL = "deepseek-v4-flash"
$env:EDGEREF_ACCESS_CODE = "your_access_code_here"
```

## User Experience

End users only need to:

1. Upload a manuscript
2. Upload source data, if available
3. Fill target journal information, if available
4. Review the Preflight Check
5. Click **Generate Review Report**
6. Download the Markdown or Word report

The page does not show API key input boxes, model selection, or provider selection.

## Developer / Debug Options

These are off by default.

```bash
export EDGEREF_ENABLE_MOCK=true
export EDGEREF_DEBUG=true
```

- `EDGEREF_ENABLE_MOCK=true`: shows a Mock Report option for local workflow testing.
- `EDGEREF_DEBUG=true`: shows the internal review prompt.

## Language Auto-Detection

The system automatically detects the manuscript language (Chinese / English / Mixed / Unknown)
based on character and word ratios, and generates the review report in the detected language.

- **English manuscript**: Review report is written in English.
- **Chinese manuscript**: Review report is written in Chinese.
- **Mixed manuscript**: Review report uses the dominant language (Chinese).
- **Unknown**: Defaults to English.

The detected language and output language are displayed in the File Processing Status
and Current Input Summary sections on the main page, and included in exported reports.

**Note**: The system only adapts the review report language. It does NOT translate the
manuscript content itself.

## Journal Catalog Database

The system supports local journal catalog matching for both international and Chinese journal directories.

This package includes a consolidated catalog file prepared from the user-provided latest directories:

- SCIE 2026-06
- SSCI 2026-06
- AHCI 2026-06
- CSSCI 2025-2026 source and extended lists
- PKU Core 2023
- CSCD 2025-2026
- AMI 2022
- Chinese Science and Technology Core 2025
- EI Compendex 2025

The main catalog file is:

```text
data/journal_catalogs/journal_catalog_master.csv
```

It contains about 29,800 catalog records after loading and deduplication. The system loads CSV files from `data/journal_catalogs/` and uses them for target journal matching. Journal catalog loading is cached during the Streamlit session to avoid repeatedly parsing the 29,000+ row catalog. A header-only template is also provided:

```text
data/journal_catalogs/chinese_core_journals_template.csv
```

### CSV Fields

```text
journal_name
normalized_name
database
database_type
version
subject_area
discipline_code
issn
eissn
cn
publisher
language
journal_code
source_file
source_sheet
source_page
notes
```

### Matching Logic

1. Exact / compact normalized name match -> Confidence: Exact
2. Aggressive normalized match -> Confidence: High
3. Substring match -> Confidence: Medium
4. Fuzzy match -> Confidence: Low
5. No local catalog hit -> Not matched

If a journal appears in multiple catalogs, the system keeps multiple matched records and displays the catalog/database summary. For example, one journal may appear in CSSCI, PKU Core, AMI and Chinese Science and Technology Core at the same time.

### Important Limitations

- A match only means the journal was found in the local CSV catalog bundled with this app.
- A non-match does **not** prove the journal is not indexed or not core. It only means it was not found in the local catalog.
- The app does **not** search the internet for journal directory information.
- Final verification should always rely on the latest official directory or the user's institution/research management office.
- Do **not** upload `.streamlit/secrets.toml` to GitHub or share it with others.

## Deployment Notes

### Journal Catalog Files

The main journal catalog file is data/journal_catalogs/journal_catalog_master.csv (about 29,800 records after loading and deduplication).
This file is bundled with the app and loaded on startup.

If you need to update the catalog:
1. Replace the CSV file(s) in data/journal_catalogs/.
2. Restart the app to clear the in-memory cache.
3. The app does **not** search the internet for journal directory information.

### Health Check (Debug Mode)

Set the environment variable EDGEREF_DEBUG=true before starting the app to enable:

- **Health Check panel** in the sidebar: verifies that key files exist and configuration is correct.
- **Show internal review prompt** checkbox.
- Health Check does NOT display API keys, access codes, or provider information.

```bash
export EDGEREF_DEBUG=true
streamlit run app.py
```

### File Upload Notes

- For first-time testing, use a copyable-text PDF, DOCX, or TXT file.
- Scanned/image-only PDFs may fail text extraction.
- Large source data files (many CSV/XLSX files or large ZIP archives) may increase parsing time.
- Uploaded files are processed temporarily in memory and are **not** saved to the server disk.

### Security Checklist

## Security Notes

- API keys are never displayed on the page.
- Real `.streamlit/secrets.toml` files must stay local or in deployment Secrets; they are not included in the distributable package.
- **File parsing cache**: Once files are processed, switching review mode, target journal info, access code, or prompt display does **not** re-parse uploaded files. Only the Reprocess files button triggers re-parsing.
- Access codes are never written to exported reports or logs.
- API keys are never written to exported reports.
- API keys should only be configured through Secrets or environment variables.
- Do not commit `.streamlit/secrets.toml`, `.env`, or any real key to GitHub.
- AI calls consume the administrator's configured API quota.
- The app does not use a database and does not persist uploaded files.

## Limitations

- Scanned-image PDFs may not be parsed unless OCR is performed before upload.
- Source data parsing is structural and does not prove data authenticity.
- AI review may miss issues or generate inaccurate statements.
- Target journal fit uses user-provided target journal information plus the local journal catalog, if available. The app does not search the internet.
